# Offline/Portable PWA Build Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make Mesozooa an installable, fully-offline home-screen PWA (precached ~67MB), and fix the
441MB `images-src` leak bloating `dist`.

**Architecture:** Move the pristine image originals out of `public/` (so Vite stops shipping them),
add `vite-plugin-pwa` (Workbox service worker + web manifest) configured to precache the whole app,
and generate the home-screen icon from the repo's `genus.svg` footprint. Verified locally via
`npm run preview` (localhost is a secure context for service workers); nginx/HTTPS deploy is a
separate follow-up.

**Tech Stack:** Vite 6, `vite-plugin-pwa` 1.3.0 (Workbox), TypeScript, `tsx` build scripts,
`rsvg-convert` (Homebrew, for one-time icon generation).

## Task 2: Generate + commit the home-screen icons

Generate the PWA icons from `genus.svg` (turquoise footprint on paper tile) and commit the PNGs so
the build has no `rsvg-convert` dependency.

**Files:**
- Create: `scripts/make-icons.ts`
- Create: `public/icons/pwa-180.png`, `pwa-192.png`, `pwa-512.png`, `pwa-512-maskable.png`

**Interfaces:**
- Consumes: `genus.svg` (repo root — a single `<g>` with one `<path>`, viewBox `0 0 239 299`).
- Produces: four committed PNG icons under `public/icons/`.

- [ ] **Step 1: Write the icon generator**

Create `scripts/make-icons.ts`. It builds a master SVG (paper `#f8f4ea` square + the `genus.svg`
footprint recolored `#0d9aa8`, centered at ~62% height) and shells out to `rsvg-convert` for each
size. The maskable variant uses more padding (~48% height) so the footprint sits inside the Android
safe zone.

```typescript
// scripts/make-icons.ts — one-time icon generator (output committed; build does NOT run this)
import { readFileSync, writeFileSync, mkdirSync } from "node:fs";
import { execFileSync } from "node:child_process";

const OUT_DIR = "public/icons";
const PAPER = "#f8f4ea";   // --bg-page
const TURQ = "#0d9aa8";    // --turq

// Extract the <g>…</g> drawing from genus.svg (skips the xml/doctype prolog) and color it turquoise.
function footprintGroup(): string {
  const raw = readFileSync("genus.svg", "utf8");
  const m = raw.match(/<g\b[\s\S]*?<\/g>/);
  if (!m) throw new Error("no <g> group found in genus.svg");
  return m[0].replace(/<path /g, `<path fill="${TURQ}" `);
}

// Master SVG at `canvas`px: paper square + footprint (native viewBox 239x299) centered at `frac` height.
function masterSvg(canvas: number, frac: number): string {
  const g = footprintGroup();
  const fh = Math.round(canvas * frac);
  const fw = Math.round((fh * 239) / 299);
  const x = Math.round((canvas - fw) / 2);
  const y = Math.round((canvas - fh) / 2);
  return (
    `<svg xmlns="http://www.w3.org/2000/svg" width="${canvas}" height="${canvas}" viewBox="0 0 ${canvas} ${canvas}">` +
    `<rect width="${canvas}" height="${canvas}" fill="${PAPER}"/>` +
    `<svg x="${x}" y="${y}" width="${fw}" height="${fh}" viewBox="0 0 239 299">${g}</svg>` +
    `</svg>`
  );
}

function render(svg: string, size: number, outfile: string): void {
  const tmp = `${OUT_DIR}/.master-${size}.svg`;
  writeFileSync(tmp, svg);
  execFileSync("rsvg-convert", ["-w", String(size), "-h", String(size), tmp, "-o", `${OUT_DIR}/${outfile}`]);
}

mkdirSync(OUT_DIR, { recursive: true });
const standard = masterSvg(1024, 0.62);
const maskable = masterSvg(1024, 0.48); // extra padding for the maskable safe zone
render(standard, 180, "pwa-180.png");
render(standard, 192, "pwa-192.png");
render(standard, 512, "pwa-512.png");
render(maskable, 512, "pwa-512-maskable.png");
console.log("icons written to", OUT_DIR);
```

## Task 3: Add `vite-plugin-pwa` (service worker + manifest)

Install the plugin and configure it to precache the whole app, exclude the gallery, and emit the
manifest. Add the apple-touch-icon link to `index.html`.

**Files:**
- Modify: `package.json` (add devDependency)
- Modify: `vite.config.ts`
- Modify: `index.html`

**Interfaces:**
- Consumes: the committed icons (Task 2); the existing two-entry build (`index.html` + `gallery.html`).
- Produces: `dist/manifest.webmanifest`, a Workbox service worker (`dist/sw.js` + precache manifest),
  registered on app load; `<link rel="apple-touch-icon">` in the served HTML.

- [ ] **Step 1: Install the plugin**

```bash
npm install -D vite-plugin-pwa@^1.3.0
```
(Do NOT install `@vite-pwa/assets-generator` — the optional peer for its own icon generator, unused
here.)

- [ ] **Step 2: Configure `VitePWA` in `vite.config.ts`**

Replace the file with:

```typescript
import { defineConfig } from "vite";
import { svelte } from "@sveltejs/vite-plugin-svelte";
import { VitePWA } from "vite-plugin-pwa";
import { resolve } from "node:path";

// Multi-page: the app (index.html) + a dev-only component gallery (gallery.html).
// The gallery is a static "testing views" surface; it ships unlinked and is NOT part of the PWA.
export default defineConfig({
  plugins: [
    svelte(),
    VitePWA({
      registerType: "autoUpdate",
      // Precache the whole app so it runs fully offline (any of ~776 genera can be the answer).
      workbox: {
        globPatterns: ["**/*.{js,css,html,woff,woff2,webp}"],
        globIgnores: ["**/gallery*", "gallery.html"], // dev-only surface — never cache
        navigateFallback: "index.html",
        navigateFallbackDenylist: [/gallery/],
        maximumFileSizeToCacheInBytes: 3 * 1024 * 1024, // GameBoard chunk is ~1MB; headroom
      },
      includeAssets: ["icons/pwa-180.png"], // apple-touch-icon (not auto-picked from manifest)
      manifest: {
        name: "Mesozooa",
        short_name: "Mesozooa",
        description: "A dinosaurs-only cladistics guessing game.",
        display: "standalone",
        start_url: "/",
        background_color: "#f8f4ea",
        theme_color: "#0d9aa8",
        icons: [
          { src: "/icons/pwa-192.png", sizes: "192x192", type: "image/png" },
          { src: "/icons/pwa-512.png", sizes: "512x512", type: "image/png" },
          { src: "/icons/pwa-512-maskable.png", sizes: "512x512", type: "image/png", purpose: "maskable" },
        ],
      },
    }),
  ],
  build: {
    rollupOptions: {
      input: {
        main: resolve(__dirname, "index.html"),
        gallery: resolve(__dirname, "gallery.html"),
      },
    },
  },
});
```
