# Local images + attribution (release sub-project A)

**Date:** 2026-07-17
**Status:** Design approved; ready for implementation plan.
**Context:** First of the pre-release sub-projects. Blocks the offline/portable build
(sub-project B) and satisfies the "per-image attribution REQUIRED before public release"
item in `docs/superpowers/deferred-findings.md` §D.

## Problem

Every specimen image is currently a **runtime network call**. 1,663 tree nodes carry an
`imageUrl` pointing at `http://commons.wikimedia.org/wiki/Special:FilePath/...`, fetched by
the browser at display time. Two consequences:

1. **Legal:** Commons files are individually licensed (CC BY / CC BY-SA / PD / occasionally
   non-free). Reuse legally requires naming author + license *per image*. The current footer
   covers only the CC0 data; the images are uncredited. This is the release blocker.
2. **Runtime dependency:** the hotlinks defeat the project's "no post-build network" goal and
   the offline-iPad goal. They're also `http://` on what will be an HTTPS site → **mixed
   content, browser-blocked** — i.e. the online path is already broken on the real deployment.

Both problems are solved by the **same Commons API call**: `extmetadata` returns the credit
fields *and* the scaled-thumbnail URL. Harvesting once gets license metadata + downloadable
bytes together.

## Decisions (settled in brainstorm)

- **Thumbnails, not originals.** Harvest ~640px-wide renditions (retina-crisp for the ~300–500px
  specimen card; originals are gigabytes and pointless). Commons renders the scaled version;
  we download that.
- **Local-always, single path.** NOT local-as-fallback. Images always load from a local path
  served by the same origin as the app. Rationale: the credit obligation exists regardless of
  which path serves the bytes, so maintaining a hotlink path buys nothing; Wikimedia discourages
  hotlinking (verified: "possible but not recommended", rate-limited); and the `http://` hotlinks
  are mixed-content-blocked under TLS anyway. One path, works identically online and offline.
  "Portable build" (sub-project B) then becomes purely a service-worker-caching concern, not an
  image-sourcing concern.
- **Ship-all-first, then measure.** This pass ships *every* image with whatever metadata Commons
  provides and emits a coverage report. The final attribution-strictness line (drop bare/non-free
  images?) is a **deliberate follow-up decision** made against real numbers — NOT part of this
  sub-project. Same instinct as the taxonomy-source doc's "one fetch + one diff before deciding."
- **Two-tier storage, matching the existing raw-pull / baked-product split.** Heavy image binaries
  are gitignored and regenerated by a fetch step (like `data/raw-taxa.json`); tiny credit strings
  are committed and folded into `tree.json` (like the baked products).
- **Presence-based skip, not mod-dates.** The photos never change (museum photos of extinct
  animals). Skip the API call + download when the file AND its credit entry already exist; a
  `--force` flag re-harvests all (e.g. when bumping thumbnail width). Mod-date/`If-Modified-Since`
  is the wrong mechanism — it still costs the roundtrip we're avoiding.

## Architecture

### 1. Harvest step — `scripts/fetch-images.ts`

New script, wired into `npm run fetch` between `fetch:pbdb` and `build:data` (it reads
`data/raw-taxa.json` for the `P18` `imageUrl` set). Own npm script `fetch:images` for
standalone runs, matching the existing per-stage pattern.

For each node with a Commons `imageUrl`, key by node id (Q-id):

- **Skip condition:** if `public/images/<Qid>.<ext>` exists AND `image-credits.json` has an entry
  for `<Qid>` → no-op (no API call, no download). `--force` bypasses.
- **Otherwise:** one Commons API call —
  `action=query&titles=File:<name>&prop=imageinfo&iiprop=extmetadata|url&iiurlwidth=640`
  (derive the `File:` title from the existing `Special:FilePath/<name>` URL). The response yields:
  - `extmetadata`: `Artist`, `LicenseShortName`, `LicenseUrl`, and the license category/flags.
  - `imageinfo[0].thumburl`: the 640px rendition URL.
- Download the thumbnail to `public/images/<Qid>.<ext>` (extension from the source file).
- Sanitize `Artist` (Commons returns raw HTML — anchors, spans) to plain text.
- Record `{ author, licenseShort, licenseUrl, licenseCategory, sourceFileUrl }` into
  `image-credits.json` keyed by Q-id.

**Politeness:** rate-limit to ~1 req/sec with a descriptive `User-Agent` (Commons sniffs it),
reusing the retry/backoff shape already in `scripts/pbdb.ts`. First full run ≈ 28 min; subsequent
runs are seconds (presence-skip).

### 2. Storage — two tiers

| Artifact | Location | Git | Analogous to |
|---|---|---|---|
| Thumbnail binaries | `public/images/<Qid>.<ext>` | **gitignored** | `data/raw-taxa.json` |
| Credit sidecar | `data/image-credits.json` | **committed** | `data/raw-*.json` (same dir/role) |
| Coverage report | `docs/superpowers/image-credits-report.md` | **committed** | — |

**Why `data/`, not `src/data/`:** `data/` is "fetch outputs that `build:data` reads and digests"
(the raw pulls). `image-credits.json` is exactly that — a network-harvest output `build:data`
consumes. It is NOT an app-imported asset (credits reach the app baked into `tree.json`), so it does
not belong in `src/data/`. Git status is orthogonal and per-file: `.gitignore` ignores the raw pulls
*by name*, so `data/image-credits.json` is tracked by default while its sibling binaries in
`public/images/` are ignored.

`public/images/` is gitignored (add `public/images/` to `.gitignore`). Vite copies `public/`
verbatim into `dist/`, so the thumbnails land next to the app and nginx serves them same-origin. A
fresh clone runs `fetch:images` to populate them — the same contract already in force for the
gitignored raw pulls.

### 3. Bake — `scripts/build-tree.ts`

`build:data` reads `image-credits.json` and, per node:

- **Rewrites `imageUrl`** from the Commons hotlink to a local path: `/images/<Qid>.<ext>`.
- **Adds credit fields** to the node: `imageAuthor`, `imageLicense`, `imageLicenseUrl`.

Type changes in `src/lib/tree/types.ts` (`TreeNode`; passthrough in `assemble.ts`) — add the three
optional credit fields. `imageUrl` stays `string?` but now holds a local path.

**Observable proof the runtime dependency is gone:** `grep wikimedia src/data/tree.json` returns
zero `imageUrl` hits post-build (the `sourceFileUrl` for provenance lives in `image-credits.json`,
not the shipped tree).

### 4. Coverage report

`fetch:images` always writes `docs/superpowers/image-credits-report.md` + a console summary as
its final step, **regardless of how many images this run actually fetched** — it reports over the
full `image-credits.json` (all accumulated entries), not just the current run's downloads. So a
presence-skip run that downloads nothing still produces a complete, current report. It breaks the
1,663 down by:

- **Completeness:** full (author + license) / partial (one missing) / bare (neither).
- **License type:** clean buckets (CC0, CC BY, CC BY-SA, PD, …) vs. **non-free / unrecognized** —
  the risk bucket.
- **Flagged list:** every file whose license category is not a known-free one, by Q-id + genus
  name, for hand review.

This pass ships all; the report only *informs* the later strictness decision.

### 5. UI credit line — `src/lib/game/components/SpecimenCard.svelte`

Fill the already-stubbed credit slot (currently hardcoded `Image: Wikimedia Commons`):

- **Full:** `{author} · {licenseShort}`, with `licenseShort` linked to `imageLicenseUrl`.
- **Partial/bare fallback:** whatever's present, falling back to `Wikimedia Commons` when author is
  absent — the slot never renders empty or broken.

Shared card → renders in both game and Explore. No layout change; slot, styling, negative-margin
tuck already exist.

## Components & boundaries

- `scripts/fetch-images.ts` — I/O harvest. Depends on `data/raw-taxa.json`; writes
  `public/images/`, `image-credits.json`, the report. Not unit-tested (I/O); verified by run.
- **Pure, TDD-tested:**
  - `sanitizeArtist(html) → string` — strips Commons `Artist` HTML to plain text.
  - `formatCredit({author, licenseShort, licenseUrl}) → display parts` — full / partial / bare
    → the strings the card renders. Table-tested.
  - Placement: `src/lib/image-credits.ts` holds both pure fns so the harvest script (imports
    `../src/lib/image-credits`) and the Svelte card (imports `../../image-credits`) share one
    formatter — one source of truth, mirroring `src/lib/game/clue.ts`. Lives under `src/lib/` (not
    `scripts/`) so the card's import stays in-tree; `scripts/` → `src/lib/` is the established
    direction (`fetch-pbdb.ts` → `../src/lib/pbdb-parse`), a `.svelte` → `scripts/` import is not.
- `build-tree.ts` — folds credits + rewrites `imageUrl` (existing bake step, extended).
- `SpecimenCard.svelte` — consumes baked node fields; renders via `formatCredit`.

## Testing & verification

- **TDD:** `sanitizeArtist`, `formatCredit` — pure, table-tested (Vitest).
- **Harvest:** run it; spot-check a sample of downloaded files + the report.
- **Build gate:** `npx tsc --noEmit` + `npx svelte-check` before commit (`verbatimModuleSyntax` is
  on — type-only imports must be `import type`; Vitest does not catch this).
- **Acceptance checks:**
  1. `grep wikimedia src/data/tree.json` → 0 `imageUrl` hits.
  2. `image-credits-report.md` exists with the three breakdowns.
  3. Specimen card renders `{author} · {license}` (license linked) for a full-credit genus, and the
     `Wikimedia Commons` fallback for a bare one, in both game and Explore.
  4. Re-running `fetch:images` without `--force` performs no downloads (presence-skip).

## Explicitly out of scope (deferred, not gaps)

- **Final attribution-strictness line** (drop bare / non-free images?) — decided later against the
  report's numbers.
- **Offline/portable build** (service worker, Add-to-Home-Screen, PWA) — sub-project B.
- **nginx deploy config** — folded into sub-project B (the iPad install path runs through it).
- **a11y keyboard nav; taxonomy/Archaeopteryx + PBDB homonyms** — separate later cycles
  (post-release for the data-quality pair, per the release decomposition).
